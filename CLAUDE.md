# ts-uitpas-sdk

## SDK Generation Pipeline

```
openapi.json  →  cleanup-openapi.js  →  openapi.clean.json  →  openapi-generator  →  sdk/
```

### Rules

- **Never edit `sdk*/`** — all files under `sdk*/`, with exception of package.json, are generated by OpenAPI Generator and will be overwritten on the next generation run.
- **Never edit `entry.json` or `uitpas.json`** — these are the upstream API specs downloaded from the UiTdatabank source. Re-download to update it.
- To fix a generation issue, either modify the openapi generation command or edit **`cleanup-openapi.js`**. This ensures the fix is applied automatically on every regeneration
- Verify the API remains consumable after making changes.

### Regenerating the SDK

```sh
# 1. (optional) refresh the spec(s)
curl -o entry.json https://stoplight.io/api/v1/projects/publiq/uitdatabank/nodes/reference/entry.json?deref=optimizedBundle

curl -o uitpas.json https://stoplight.io/api/v1/projects/publiq/uitpas/nodes/reference/uitpas.json?deref=optimizedBundle

# 2. clean the spec with the
node cleanup-openapi.js entry.json
node cleanup-openapi.js uitpas.json

# 3. generate TypeScript sources
npx @openapitools/openapi-generator-cli generate \
  -i entry.clean.json \
  -g typescript-fetch \
  -o ./sdk-entry
```

```
npx @openapitools/openapi-generator-cli generate \
  -i uitpas.clean.json \
  -g typescript-fetch \
  -o ./sdk-uitpas
```

### What cleanup-openapi.js does

1. **Schema renames** — renames top-level schema names that contain characters invalid for code generation (e.g. `Event-@id` → `EventId`) and updates all `$ref` pointers via raw string replacement.
2. **Image.@id removal** — removes the `@id` property from the `Image` schema. OpenAPI Generator strips `@` from property names, so `@id` would become TypeScript `id`, conflicting with the Image schema's existing `id` (UUID) property. Removing `@id` eliminates the duplicate identifier; the UUID `id` field is sufficient to reference images in all API calls.
